# 프로젝트 개요

## 기술 스택
- 백엔드: FastAPI
- 프론트엔드: React
- DB: PostgreSQL
- AI: Groq llama-3.3-70b-versatile

## 구현 우선순위
- Phase 1: 인증 + 일기 CRUD + 캘린더 (AI 없이 동작)
- Phase 2: 자정 배치(분석) + 대시보드 그래프
- Phase 3: 페르소나(초기 분류 + 재분류) + AppBar 캐릭터
- Phase 4: 아침 배치(미션 생성) + 미션 슬롯 UI
- Phase 5: 디테일(애니메이션, 그래프 마커, 오류 처리)

각 Phase는 독립적으로 배포 가능한 단위.

## AI 호출 정책

| 호출 항목 | 빈도 | 트리거 |
|----------|------|--------|
| 자기효능감 분석 | 하루 1회 | 자정에 날짜 변경 시 |
| 미션 생성 | 하루 최대 1회 | 매일 아침, 슬롯 3개 미만일 때만 |
| 페르소나 재분류 | 일기 15개당 1회 | 누적 카운터 도달 시 |

- 한 사용자 하루 AI 호출 상한: 2회 (페르소나 재분류 날은 +1회)
- 모든 AI 응답은 DB에 저장하여 재사용

## 제약

**Do**
- AI 호출 결과는 반드시 DB에 저장 후 재사용
- AI 호출 횟수는 위 정책 안에서만
- 사용자가 직접 입력한 정보(성적 등)는 알고리즘 입력값으로 적극 활용

**Don't**
- AI 연속 호출 금지
- 미션 3개 초과 생성 금지
- 일기 수정 시마다 재분석 금지 (분석은 날짜 변경 시 1회)
